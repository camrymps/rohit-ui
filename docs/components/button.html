Button

The Button component provides a classic Windows 98-style button with various states and options.

Import

import { Button } from 'rohit-ui';

Props

Name Type Default Description
children ReactNode - Content to display in the button
primary boolean false Makes the button a primary button with themed styling
disabled boolean false Whether the button is disabled
icon ReactNode - Icon to display before the button text
iconPosition 'left' | 'right' 'left' Position of the icon relative to the text
size 'small' | 'medium' | 'large' 'medium' Size of the button
fullWidth boolean false Makes the button take up the full width of its container
onClick (event: React.MouseEvent) => void - Function to call when the button is clicked
rohitMode boolean false Enables Rohit Mode with silly animations and behaviors

CSS Customization

You can customize the Button component using styled-components:

import styled from 'styled-components';
import { Button } from 'rohit-ui';

const CustomButton = styled(Button)`
  background: linear-gradient(90deg, var(--win98-silver), white);
  border-width: 3px;
  border-style: dashed;
  
  &:hover {
    background: linear-gradient(90deg, white, var(--win98-silver));
  }
`;

function MyComponent() {
  return (
    
      Custom styled button
    
  );
}

Rohit Mode Behavior

When Rohit Mode is enabled, the Button component will:

  • Spin randomly when hovered
  • Change colors unpredictably
  • Sometimes move away from the cursor when being clicked
  • Occasionally make Windows 98 error sounds
  • Take longer than necessary to perform simple actions

Rohit Mode can be enabled per-component or globally:
